### Building on What You Know

In the Foundations level, you learned:
- **Personal security**: Password managers, MFA, phishing detection
- **Core concepts**: CIA triad, encoding, hashing vs encryption
- **Basic threats**: Social engineering, malware, breach patterns
- **Network fundamentals**: DNS, HTTPS, attack surfaces

This Applied level deepens those concepts for **team and organizational security**. You'll learn:
- How to apply threat modeling to real systems
- How vulnerabilities connect into attack chains
- How authentication works at scale (JWT, OAuth2, sessions)
- How to detect and respond to incidents using logs
- How to make risk-based security decisions

**Key Shift**: From "How do I stay safe?" to "How do we build systems that resist attacks?"

## What Changes from Foundations to Applied?

### 1. Scale: Individual ‚Üí Team/Organization

**Foundations Example**: "I use a password manager for my 20 accounts"

**Applied Example**: "How do we enforce MFA for 500 employees across 15 applications with SSO?"

### 2. Complexity: Single Control ‚Üí Systems Thinking

**Foundations Example**: "Strong passwords resist brute force"

**Applied Example**: "How do we handle password resets securely? What if email is compromised? What's the account recovery flow? Where do we log failed attempts?"

### 3. Responsibility: Personal Safety ‚Üí Protecting Others

**Foundations Example**: "I detect phishing to protect myself"

**Applied Example**: "How do I configure email filters to protect 1,000 users? What's the incident response if someone clicks?"

### 4. Trade-offs: Best Practice ‚Üí Risk-Based Decisions

**Foundations Example**: "Always use the strongest encryption"

**Applied Example**: "Encryption adds latency. For this system, do we need encryption at rest? In transit? Both? What's the business impact of each choice?"

<Callout variant="insight" title="The Applied Mindset">
**Foundations asks**: "What's the right thing to do?"

**Applied asks**: "What's the right thing to do *in this context*, with *these constraints*, for *this risk level*, and how do we *verify it's working*?"
</Callout>

## Career Paths: Where Applied Skills Lead

### Security Analyst
**What you do**: Monitor alerts, investigate incidents, recommend fixes
**Applied skills used**: Log analysis, threat modeling, vulnerability assessment
**Typical journey**: Junior analyst (Foundations) ‚Üí Analyst (Applied) ‚Üí Senior analyst (Practice)

### Application Security Engineer
**What you do**: Review code for vulnerabilities, implement secure authentication, run security testing
**Applied skills used**: OWASP Top 10, session management, secure SDLC
**Typical journey**: Developer with security interest ‚Üí AppSec engineer (Applied) ‚Üí AppSec lead (Practice)

### SOC Analyst (Security Operations Center)
**What you do**: Monitor SIEM, triage alerts, escalate incidents, create detections
**Applied skills used**: Log analysis, attack chain recognition, incident timelines
**Typical journey**: IT support ‚Üí SOC Tier 1 (Foundations) ‚Üí SOC Tier 2 (Applied) ‚Üí SOC Tier 3 / Threat Hunter (Practice)

### Security Consultant
**What you do**: Assess client security, conduct audits, recommend improvements
**Applied skills used**: Threat modeling, risk assessment, control recommendations
**Typical journey**: Technical role ‚Üí Junior consultant (Applied) ‚Üí Senior consultant (Practice)

### Cloud Security Specialist
**What you do**: Secure cloud infrastructure, implement IAM, configure security services
**Applied skills used**: Network segmentation, identity architecture, secure configuration
**Typical journey**: Cloud engineer ‚Üí Cloud security (Applied) ‚Üí Cloud security architect (Practice)

## Study Recommendations

### Time Commitment
- **Estimated**: 18 CPD hours total
- **Suggested pace**: 2-3 hours per week = 6-9 weeks
- **Intensive option**: 3-4 hours per week = 4-6 weeks

### Learning Approach

**1. Active Practice (Not Passive Reading)**
- Complete every interactive tool
- Write out threat models by hand first, then use the tools
- Pause videos to predict what happens next

**2. Connect to Foundations**
- When you see "JWT", recall "Foundations: hashing vs encryption"
- When you see "lateral movement", recall "Foundations: Target breach via HVAC vendor"
- Each Applied concept deepens a Foundations concept

**3. Apply to Your Context**
- If you work in development: Focus on Module 3 (vulnerabilities) and Module 4 (auth)
- If you work in operations: Focus on Module 5 (detection) and Module 6 (risk)
- If you're transitioning careers: Complete everything in order

**4. Use the Case Studies**
- Real breaches teach more than hypotheticals
- For each breach: Draw the attack chain, identify failed controls, propose fixes

### Success Metrics

You're ready for Practice (Advanced) level when you can:
- ‚úÖ Threat model a system in 30 minutes using STRIDE
- ‚úÖ Explain 5+ OWASP Top 10 vulnerabilities with examples
- ‚úÖ Design secure authentication with JWT + refresh tokens
- ‚úÖ Investigate an incident by analyzing logs
- ‚úÖ Build risk matrices and prioritize vulnerabilities
- ‚úÖ Explain security trade-offs to non-technical stakeholders
