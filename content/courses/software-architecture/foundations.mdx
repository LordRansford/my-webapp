## Mindset and roles

<SectionProgressToggle courseId="software-architecture" levelId="foundations" sectionId="soft-arch-foundations-mindset-roles" />

Software development is a team sport. The work is shared across developers, testers, product people, operations, and security. Everyone owns part of the outcome, even if their tools look different.

<GlossaryTip term="architecture">
The structure of a system, the responsibilities inside it, and the decisions that are costly to change later.
</GlossaryTip>

<Callout variant="concept" type="info">
Good architecture makes teamwork easier by making decisions and boundaries visible.
</Callout>

## Architecture building blocks

<SectionProgressToggle courseId="software-architecture" levelId="foundations" sectionId="soft-arch-foundations-building-blocks" />

Systems are built from components and services. A <GlossaryTip term="component">A unit of responsibility such as a module, service, or external system.</GlossaryTip> owns data and behaviour. An <GlossaryTip term="API">A defined way for systems to request and exchange data.</GlossaryTip> is the contract between them.

<GlossaryTip term="monolith">
A single deployed system where most responsibilities live together.
</GlossaryTip>

<GlossaryTip term="microservice">
A small service that owns one clear job and can be deployed on its own.
</GlossaryTip>

## Code design basics

<SectionProgressToggle courseId="software-architecture" levelId="foundations" sectionId="soft-arch-foundations-code-design" />

Clean code is about readability, not clever tricks. A <GlossaryTip term="dependency">A thing your code relies on to do its job.</GlossaryTip> should be obvious. Tests are the safety net. A <GlossaryTip term="test double">A fake or simplified version of a dependency used in tests.</GlossaryTip> keeps tests fast and focused.

<Callout variant="concept" type="info">
If the next person can explain your code in a sentence, you are doing it right.
</Callout>

## Delivery, security and reliability basics

<SectionProgressToggle courseId="software-architecture" levelId="foundations" sectionId="soft-arch-foundations-delivery-and-safety" />

Software lives across environments like dev, test, staging, and production. <GlossaryTip term="continuous integration">A practice where changes are merged often and tested automatically.</GlossaryTip> keeps the codebase stable. <GlossaryTip term="least privilege">Only give access that is required, no more.</GlossaryTip> keeps risk low.
