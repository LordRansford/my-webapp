# Screenshot guidance (manual submission support)

Do not automate screenshot capture.
Provide clear, legible screenshots that demonstrate structure, evidence capture, and accessibility.

## Recommended screenshots

- Course overview page
  - shows the course structure and progression
- A foundations level page
  - includes learning objectives and estimated hours
- A tool interaction example
  - input area and an example output, with the tool limitations visible if applicable
- A quiz example
  - formative quiz block and completion behaviour
- Progress tracking example
  - CPD time tracker and completion indicator behaviour
- Accessibility example
  - visible focus state, keyboard navigation, or skip-to-content behaviour

## Notes for assessors

- Avoid screenshots that contain personal data.
- Use a consistent browser zoom level and window size for readability.
- If a page includes dynamic content, capture both the initial state and a completed state.

# Screenshot guidance (manual submission)

Do not automate capture. Provide clean screenshots taken from a standard desktop browser window.

## Recommended screenshots

### Platform and structure

- Home page showing the overall positioning
- Course landing page for each CPD course
- A level page showing learning objectives and estimated hours

### Learning interaction and assessment

- A tool interaction example showing inputs and structured outputs
- A quiz example showing the quiz block and completion behaviour
- A completion indicator example (section progress toggle)

### CPD tracking and evidence

- A level page showing time tracking input
- The CPD page showing course hour totals and progression
- The evidence page showing evidence text and activity log

### Accessibility

- A screenshot showing the skip to content link focused
- A screenshot showing visible focus styles on navigation

## Notes

- Remove any personal data before submission.
- Use consistent browser zoom and window size across screenshots.
