# CPD readiness checklist (internal gate)

Use this checklist before preparing a submission to any CPD body.

## Course completeness

- [ ] Cybersecurity: foundations, applied, practice, summary and games present
- [ ] AI: foundations, intermediate, advanced, summary and games present
- [ ] Software Architecture: foundations, intermediate, advanced, summary and games present
- [ ] Data: foundations, intermediate, advanced, summary and games present
- [ ] Digitalisation: foundations, intermediate, advanced, summary and games present

## Learning objectives clarity

- [ ] Objectives use action verbs and are measurable
- [ ] Foundations objectives are knowledge-focused
- [ ] Intermediate objectives are application-focused
- [ ] Advanced objectives are analysis and judgement-focused
- [ ] Summary and games reference existing objectives only

## Time tracking

- [ ] Each level page includes an estimated hours field
- [ ] Hours are conservative and defensible
- [ ] Time entry is visible and recorded per level

## Assessment presence

- [ ] Quizzes exist and are formative
- [ ] Completion indicators exist for sections
- [ ] Tools and dashboards provide practice artefacts

## Evidence capture and extraction

- [ ] Evidence can be generated in a structured format server-side
- [ ] Evidence includes course title, learner identifier, hours, objectives, completion date
- [ ] Evidence does not claim certification

## Accessibility

- [ ] Keyboard navigation works end to end
- [ ] Focus states are visible
- [ ] Contrast is acceptable for body text
- [ ] Diagrams have supporting text explanation

## Update policy

- [ ] Content changes are version-controlled and reviewable
- [ ] Significant changes have an audit note
